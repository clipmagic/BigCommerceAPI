BigCommerce API developer information

This module is a wrapper for the BigCommerce API and allows you to make calls to your store and display the results on your ProcessWire site

Learn more about BigCommerce at www.bigcommerce.com and the BigCommerce API at https://developer.bigcommerce.com/api.

Prerequisites

  • You must have a BigCommerce store already set up.
  • You must know your API login credentials. These can be:
    • BasicAuth
      • store name: This MUST be the original store name issued by BigCommerce, without an ending backslash, eg:
        https://store-xxxxxx.mybigcommerce.com
        where xxxxx is the unique store id issued by BigCommerce.
      • username: This is the username you create in BigCommerce for the Legacy API. To create the username go to Set Up & Tools »Legacy API accounts in the BigCommerce store admin.
      • api_key: After creating the Legacy API user, copy&paste the API token in to the ProcessWire module BigCommerceAPI API key field.
      • Save your BigCommerceAPI module settings.
    • OAuth
      • client_id
      • auth_token
      • store_hash
    Note: Enter info into only one of the two login fieldsets. The module will default to the BasicAuth method.

Usage

Firstly, in your template instantiate the module as it is not autoloaded, eg:

$bc = $modules->get('BigCommerceAPI');.

Then in your PW template, you can call any BigCommerce API function within the Bigcommerce\Api\Client namespace with, eg

$product = $bc->api('getProduct', 124);

where the first parameter is the BigCommerce API function name and subsequent parameters are the function arguments.

Have fun with it.
